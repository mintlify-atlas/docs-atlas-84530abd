## Endpoint

```http
GET /v1/{organizationId}/posts
```

Retrieves a paginated list of posts for the specified organization. Supports filtering by status and content type, as well as sorting by creation date.

## Authentication

This endpoint requires Bearer token authentication. Include your API key in the Authorization header:

```http
Authorization: Bearer YOUR_API_KEY
```

## Path Parameters

<ParamField path="organizationId" type="string" required>
  The unique identifier of the organization. The API key must have access to this organization.
  
  Example: `org_123`
</ParamField>

## Query Parameters

<ParamField query="limit" type="integer" default="10">
  Number of items per page. Must be between 1 and 100.
  
  Example: `10`
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number to retrieve. Must be at least 1.
  
  Example: `1`
</ParamField>

<ParamField query="sort" type="string" default="desc">
  Sort order by creation date.
  
  Options: `asc` | `desc`
</ParamField>

<ParamField query="status" type="string[]" default="[published]">
  Filter posts by status. Can be a single value or multiple values by repeating the query parameter.
  
  Options: `draft` | `published`
  
  Default: `["published"]` (only published posts)
  
  Examples:
  - Single value: `?status=published`
  - Multiple values: `?status=draft&status=published`
</ParamField>

<ParamField query="contentType" type="string[]">
  Filter posts by content type. Can be a single value or multiple values by repeating the query parameter.
  
  Options: `changelog` | `linkedin_post`
  
  Default: All content types
  
  Examples:
  - Single value: `?contentType=changelog`
  - Multiple values: `?contentType=changelog&contentType=linkedin_post`
</ParamField>

## Response

<ResponseField name="posts" type="array">
  Array of post objects matching the query criteria.
  
  <Expandable title="Post Object">
    <ResponseField name="id" type="string">
      Unique identifier for the post.
    </ResponseField>
    
    <ResponseField name="title" type="string">
      The title of the post.
    </ResponseField>
    
    <ResponseField name="content" type="string">
      The content of the post in its original format.
    </ResponseField>
    
    <ResponseField name="markdown" type="string">
      The markdown representation of the post content.
    </ResponseField>
    
    <ResponseField name="contentType" type="string">
      The type of content. Options: `changelog` | `linkedin_post`
    </ResponseField>
    
    <ResponseField name="sourceMetadata" type="object | null">
      Additional metadata about the source of the post.
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Publication status. Options: `draft` | `published`
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp when the post was created.
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp when the post was last updated.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  Pagination metadata for navigating through the result set.
  
  <Expandable title="Pagination Object">
    <ResponseField name="limit" type="integer">
      Number of items per page.
    </ResponseField>
    
    <ResponseField name="currentPage" type="integer">
      The current page number.
    </ResponseField>
    
    <ResponseField name="nextPage" type="integer | null">
      The next page number, or `null` if this is the last page.
    </ResponseField>
    
    <ResponseField name="previousPage" type="integer | null">
      The previous page number, or `null` if this is the first page.
    </ResponseField>
    
    <ResponseField name="totalPages" type="integer">
      Total number of pages available.
    </ResponseField>
    
    <ResponseField name="totalItems" type="integer">
      Total number of items across all pages.
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

<CodeGroup>
```bash cURL
curl -X GET "https://api.notra.ai/v1/org_123/posts?limit=10&page=1&sort=desc" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```bash Filter by Status
curl -X GET "https://api.notra.ai/v1/org_123/posts?status=published&limit=10" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```bash Multiple Filters
curl -X GET "https://api.notra.ai/v1/org_123/posts?status=draft&status=published&contentType=changelog&limit=20" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript/TypeScript
const response = await fetch(
  'https://api.notra.ai/v1/org_123/posts?limit=10&page=1&sort=desc',
  {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY'
    }
  }
);

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.get(
    'https://api.notra.ai/v1/org_123/posts',
    params={
        'limit': 10,
        'page': 1,
        'sort': 'desc'
    },
    headers={
        'Authorization': 'Bearer YOUR_API_KEY'
    }
)

data = response.json()
print(data)
```

## Status Codes

| Code | Description |
|------|-------------|
| 200  | Posts fetched successfully |
| 400  | Invalid path params or query parameters |
| 401  | Missing or invalid API key |
| 403  | Forbidden - API key does not have access to the specified organization |
| 503  | Authentication service unavailable |

## Notes

- The default behavior is to return only `published` posts unless you explicitly specify the `status` query parameter
- To retrieve all posts regardless of status, include both values: `?status=draft&status=published`
- Posts are ordered by creation date with the most recent first by default (`sort=desc`)
- The maximum page size is 100 items
- The API key must belong to the organization specified in the path, otherwise a 403 error will be returned
